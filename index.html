About

I'm a PhD candidate in Computer Science at Case Western Reserve University, specializing in AI infrastructure, distributed ML systems, and eBPF-based observability. PRISM – Priority-Aware Transport for Distributed ML Training
Priority-based selective reliability transport between NCCL/Gloo and UDP; classifies gradients by magnitude — full retransmission for high-priority, bounded retries + local compensation for low-priority; dual congestion windows with adaptive window borrowing; GPU-accelerated TopK selection, SACK-based packet tracking, kernel-bypass (LibVMA) and GSO/GRO optimizations; achieves 8–15% throughput gain for CNNs and 15–21% for LLMs over TCP under packet loss, on 32 GPUs (OSC) and cross-site FABRIC testbed; matches NDP latency at 144-worker scale on commodity infrastructure.

Priority-Based eBPF Network Flow Telemetry for AI / DML Infrastructure
In-kernel eBPF priority-aware sketch framework with multi-layer architecture: priority table (precise per-packet tracking) + heavy flow table + 3-layer Count-Min Sketch; sequence-number-based retransmission detection with configurable thresholds; atomic operations (__sync_fetch_and_add) for thread-safe concurrent updates; voting-based hash collision resolution and CMS minimum-value flow reconstruction; achieves 96% Top-K accuracy, 100% priority-packet recall, and <1.1% throughput overhead at 10 Gbps on FABRIC testbed with CAIDA 2019 backbone traffic replay.

High-Resolution eBPF System Resource Profiling for DML Workloads
Kernel-level eBPF observability via dual HashPipe pipelines: memory probes on 12 libc/kernel allocation APIs (malloc, free, munmap) into ptr2size + priority_vm BPF maps; CPU via sched_switch tracepoint + bpf_get_current_pid_tgid(). HashPipe's O(1) per-event cascading stages (vs O(N log N) sort-based Top-K) are physically unrolled via #pragma unroll for eBPF verifier compliance, with atomic_fetch_and_add() for concurrent-safe map updates; achieves 10–14× finer temporal resolution than top/htop at ~11 ms latency with >90% Top-K fidelity.

Scotch – Elastic SDN Overlay for Control-Plane Scalability
Exploits the asymmetry between physical switches (high data-plane throughput, weak OFA CPUs) and commodity vSwitches (strong CPUs): new flows elastically tunneled to a fully connected vSwitch mesh when Packet-In rate signals congestion. Per-ingress-port queues with round-robin scheduling share controller service rate evenly; OpenFlow Group Table (ECMP) load-balances across vSwitches; three-level priority queue (admitted → large flow → ingress-port) governs precedence; Flow Info DB maps tunnel ID → physical switch for migrating stable large flows back to the physical network. Validated on a 34-node GENI Clos topology; reduced controller congestion by >70% with elastic scale-down when attacks subside.

BiLSTM-Based Side-Channel Modeling for Distributed ML Analysis
Training-phase black-box attack on MXNet BSP distributed training reconstructing full DNN architectures via two unprivileged side channels: Intel RAPL PP0 + DRAM power sampled at 50μs via powercap/sysfs, and 0-payload network packet sequences (Seq_T) for tensor size extraction. Lookahead (sliding-window SMA) + Lookbehind algorithms isolate forward passes; band-pass filtering segments individual layers. Decision Tree (entropy, 97.7%, 10-fold CV on 84K+ FC/Conv segments) and BiLSTM classify layer types; a weighted-average ensemble of top-5 MLP regressors (weights ∝ 1/validation error) recovers Conv kernel dimensions and FC neuron counts. DT/MLP achieves >97% layer-type accuracy vs. BiLSTM's ~92% (MAIA baseline); validated across 20 DNN models (LeNet through VGG/ResNet families).
